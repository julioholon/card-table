# Issue Tracker

Issues are tracked as local Markdown files under `.scratch/` in the repo root.

## Convention

Each issue is a file: `.scratch/issues/{id}-{slug}.md`

## Workflow

- New issues start as `needs-triage`
- Move to `ready-for-agent` when fully specified
- Move to `ready-for-human` when it needs human implementation
- Move to `wontfix` when explicitly not actioned
- Move to `needs-info` when waiting on reporter
